Permasalahan Umum dalam Kriptografi File

Sebelum memahami SimpleCipher v3, penting untuk mengenal celah keamanan yang umum ditemukan pada implementasi enkripsi yang naif.

🔑

Password Langsung Jadi Kunci

Banyak implementasi menggunakan password pengguna secara langsung sebagai kunci enkripsi. SHA-256 tunggal terlalu cepat — GPU modern bisa mencoba >10 miliar kombinasi/detik, membuat password pendek tidak aman.

Brute-Force Vulnerability
🔁

Enkripsi Deterministik

Tanpa nilai acak per sesi (IV/Nonce), plaintext yang sama + password yang sama selalu menghasilkan ciphertext yang identik. Penyerang dapat mendeteksi duplikasi data atau melakukan known-plaintext attack.

Pattern Analysis Attack
📐

Operasi Linear Murni

Cipher yang hanya mengandalkan XOR dan operasi aritmatika dapat direpresentasikan sebagai sistem persamaan linear. Penyerang dapat mengeksploitasi ini melalui linear cryptanalysis untuk merekonstruksi kunci.

Linear Cryptanalysis
🚫

Tidak Ada Verifikasi Integritas

Enkripsi hanya menjamin kerahasiaan, bukan keaslian. Tanpa autentikasi, penyerang dapat memodifikasi ciphertext dan mengamati pesan error untuk membocorkan plaintext secara bertahap — disebut padding oracle attack.

Chosen-Ciphertext Attack
💾

Pemrosesan File Besar Tidak Efisien

Memuat seluruh file ke RAM sebelum mengenkripsi tidak skalabel. File berukuran beberapa gigabyte dapat menyebabkan Out-of-Memory (OOM) pada perangkat dengan RAM terbatas.

Memory Inefficiency

01

Inisialisasi & Key Derivation

PBKDF2 IV

Password tidak pernah digunakan langsung sebagai kunci. Pertama, dua nilai acak 16 byte dihasilkan: IV (untuk keunikan ronde) dan Salt (untuk PBKDF2). Salt diproses bersama password melalui PBKDF2-HMAC-SHA256 dengan 10.000 iterasi menghasilkan master_key. Dari master_key diturunkan mac_key untuk autentikasi — ini disebut domain separation, memastikan kunci enkripsi dan autentikasi tidak saling berhubungan.

pw   = password.encode('utf-8') if isinstance(password, str) else password
iv   = os.urandom(IV_SIZE)    # 16 byte acak per sesi — IV
salt = os.urandom(SALT_SIZE)  # 16 byte acak per sesi — Salt PBKDF2

# PBKDF2: 10.000 iterasi memperlambat brute-force GPU ~1.000×
master_key = hashlib.pbkdf2_hmac(
    'sha256', pw, salt, PBKDF2_ITER, dklen=32
)

# Domain separation: mac_key berbeda dari kunci enkripsi
mac_key = hashlib.sha256(master_key + b'MAC').digest()
02

Padding PKCS#7 & Pembagian Chunk

Memory-Efficient

Data asli ditambahkan padding agar panjangnya kelipatan 16 byte (PKCS#7). Nilai setiap byte padding sama dengan jumlah byte yang ditambahkan — ini memungkinkan penghapusan deterministik saat dekripsi. Data kemudian dibagi menjadi potongan maksimal 4 MB untuk diproses secara berurutan, sehingga RAM yang terpakai selalu terbatas.

Data 13B:
H e l l o … (13 byte data)
03 03 03
= 16B
Data 16B:
data (16 byte)
10 10 … 10 (16 byte)
= 32B
def _add_padding(data: bytes) -> bytes:
    # PKCS#7: nilai padding = jumlah byte yang ditambahkan
    pad = 16 - (len(data) % 16)
    return data + bytes([pad] * pad)

padded = _add_padding(plaintext)

# Chunking: proses per 4 MB, RAM tetap terbatas
for ci, start in enumerate(range(0, len(padded), CHUNK_SIZE)):

03

Prekomputasi Round Key & Round Salt

Key Derivation

Untuk setiap chunk, fungsi _precompute() menghasilkan 6 round key dan 6 round salt — masing-masing sepanjang n byte (ukuran chunk). Material ini diturunkan dari master_key + iv + tag + nomor ronde via SHA-256 chaining. Parameter chunk_idx memastikan setiap chunk mendapat material yang berbeda. 

def _gen_bytes(master_key, iv, tag, index, n):
    # SHA-256 chaining: h₀ → h₁ → h₂ → … dipotong sepanjang n
    h   = hashlib.sha256(master_key + iv + tag + index.to_bytes(4,'big')).digest()
    out = bytearray()
    while len(out) < n:
        out += h
        h = hashlib.sha256(h + master_key).digest()  # chain
    return bytes(out[:n])

def _precompute(master_key, iv, chunk_idx, n):
    # Tag 'RK...' untuk round key, 'ST...' untuk round salt
    rks   = [_gen_bytes(master_key, iv, b'RK'+chunk_idx.to_bytes(4,'big'), r, n)
             for r in range(ROUNDS)]  # 6 round key
    salts = [_gen_bytes(master_key, iv, b'ST'+chunk_idx.to_bytes(4,'big'), r, n)
             for r in range(ROUNDS)]  # 6 round salt
    return rks, salts
04

6 Ronde SPN — Inti Enkripsi

SPN Core

Setiap chunk melewati 6 ronde transformasi berurutan. Detail setiap ronde dijelaskan pada bagian berikutnya. 

def _enc_chunk(raw: bytes, rks, salts) -> bytes:
    data = np.frombuffer(raw, dtype=np.uint8).copy()
    for r in range(ROUNDS):          # Ronde 0 → 1 → 2 → 3 → 4 → 5
        data = _xor(data, rks[r])    # 1. XOR dengan round key
        data = _apply_sbox(data)     # 2. S-Box: substitusi non-linear
        data = _add_salt(data,       # 3. AddSalt: penjumlahan mod 256
                         salts[r])
        data = _swap_halves(data)    # 4. Swap: tukar dua setengah
    return data.tobytes()
05

Autentikasi HMAC & Susun Output Final

HMAC Encrypt-then-MAC

Setelah semua chunk terenkripsi, HMAC-SHA256 dihitung atas seluruh konten yang akan disimpan: VERSION + IV + SALT + ciphertext. Pola Encrypt-then-MAC memastikan penyerang tidak dapat memanipulasi apapun di file .sc tanpa terdeteksi. Output final disusun dengan IV dan SALT di header sehingga dekripsi dapat merekonstruksi semua kunci yang dibutuhkan. 

ciphertext_body = bytes(out)  # gabungan semua chunk terenkripsi

# HMAC atas VERSION + IV + SALT + seluruh ciphertext
mac_tag = hmac.new(
    mac_key,
    VERSION + iv + salt + ciphertext_body,
    hashlib.sha256
).digest()

# Format output: [1B] + [16B] + [16B] + [?B] + [32B]
return VERSION + iv + salt + ciphertext_body + mac_tag
Core Algorithm

Detail 4 Operasi dalam Setiap Ronde

Setiap dari 6 ronde menjalankan 4 operasi berurutan. Kombinasi operasi linear dan non-linear mencapai Confusion dan Diffusion Shannon.

XOR dengan Round Key

Setiap byte data di-XOR dengan byte round key yang bersesuaian. XOR bersifat self-inverse: (a ⊕ k) ⊕ k = a, sehingga operasi dekripsi identik dengan enkripsi — cukup XOR dengan kunci yang sama. Ini mencampur kunci ke dalam data (konfusi).

Data: 01001000 (= 72)
Key: 10110101 (= 181)
XOR: 11111101 (= 253) 
# Versi numpy (cepat — operasi array level C)
def _xor(data: np.ndarray, key: bytes) -> np.ndarray:
    return data ^ np.frombuffer(key, dtype=np.uint8)

# Versi pure Python (fallback)
def _xor(data: bytearray, key: bytes) -> bytearray:
    return bytearray(a ^ b for a, b in zip(data, key))

S-Box — Substitusi Non-Linear

Setiap byte diganti menggunakan lookup table (S-Box) berukuran 256 entri. S-Box dibuat dengan algoritma Fisher-Yates shuffle menggunakan seed tetap (42), memastikan S-Box yang sama di setiap eksekusi. S-Box bersifat bijektif — Inverse S-Box diturunkan otomatis untuk dekripsi.

Ini adalah satu-satunya operasi non-linear dalam cipher, memenuhi prinsip confusion Shannon dan mencegah analisis linear/diferensial.

Input byte→SBOX[input]
0x00 (0)→? (nilai acak)
0x01 (1)→? (nilai berbeda)
0xFF (255)→? (bijektif)
# Inisialisasi S-Box (sekali saat program start)
_rng  = random.Random(42)      # seed tetap
SBOX  = list(range(256))
_rng.shuffle(SBOX)             # Fisher-Yates shuffle

# Inverse S-Box untuk dekripsi
INV_SBOX = [0] * 256
for i, v in enumerate(SBOX):
    INV_SBOX[v] = i            # jika SBOX[i]=j, INV_SBOX[j]=i

# Versi numpy (O(n) di level C — sangat cepat)
_NP_SBOX     = np.array(SBOX,     dtype=np.uint8)
_NP_INV_SBOX = np.array(INV_SBOX, dtype=np.uint8)

def _apply_sbox(data: np.ndarray) -> np.ndarray:
    return _NP_SBOX[data]      # lookup seluruh array sekaligus

def _apply_inv_sbox(data: np.ndarray) -> np.ndarray:
    return _NP_INV_SBOX[data]

AddSalt — Penjumlahan Modulo 256

Setiap byte data ditambah dengan byte round salt menggunakan aritmatika modulo 256. Berbeda dengan XOR, penjumlahan dapat menyebabkan carry propagation — perubahan 1 bit dapat mempengaruhi bit-bit yang lebih tinggi, meningkatkan efek difusi.

Data: 200
Salt: 80
Hasil: (200 + 80) % 256 = 24
Inverse: (24 - 80) % 256 = 200 ✓ 
# Enkripsi: tambah salt (mod 256)
def _add_salt(data: np.ndarray, salt: bytes) -> np.ndarray:
    return (
        (data.astype(np.uint16) +            # upcast ke uint16 cegah overflow
         np.frombuffer(salt, dtype=np.uint8)
        ) % 256
    ).astype(np.uint8)

# Dekripsi: kurangi salt (mod 256) — self-inverse dengan operasi terbalik
def _sub_salt(data: np.ndarray, salt: bytes) -> np.ndarray:
    return (
        (data.astype(np.uint16) -
         np.frombuffer(salt, dtype=np.uint8)
        ) % 256
    ).astype(np.uint8)

SwapHalves — Difusi Spasial

Data dibagi tepat di tengah, lalu kedua bagian ditukar posisinya: [A | B] → [B | A]. Operasi ini bersifat self-inverse — swap dua kali mengembalikan data ke posisi semula. Tujuannya adalah menyebarkan pengaruh byte dari bagian pertama ke bagian kedua dan sebaliknya (diffusion).

A
setengah pertama
B
setengah kedua
⇄ swap
B
setengah kedua
A
setengah pertama
# Versi numpy
def _swap_halves(data: np.ndarray) -> np.ndarray:
    mid = len(data) // 2
    return np.concatenate([data[mid:], data[:mid]])
    # [A|B] → data[mid:] = B, data[:mid] = A → [B|A]

# Versi pure Python
def _swap_halves(data: bytearray) -> bytearray:
    mid = len(data) // 2
    return data[mid:] + data[:mid]

# Self-inverse: swap dua kali = kembali ke semula
# [A|B] → swap → [B|A] → swap → [A|B] ✓

Alur Dekripsi — Verifikasi Dulu, Dekripsi Kemudian

Urutan dekripsi dirancang mengikuti standar Encrypt-then-MAC: HMAC diverifikasi sebelum ciphertext disentuh, mencegah serangan padding oracle.

①

Validasi Format & VERSION

File harus memiliki panjang minimum 65 byte (1+16+16+32). Byte pertama harus 0x03 (VERSION v3). File dari versi lama (v1/v2) ditolak eksplisit dengan pesan informatif.

MIN_LEN = 1 + IV_SIZE + SALT_SIZE + MAC_SIZE  # = 65 byte
if len(ciphertext) < MIN_LEN:
    raise ValueError("File tidak valid atau bukan format SimpleCipher v3.")
if ciphertext[:1] != VERSION:  # VERSION = b'\x03'
    raise ValueError("Format tidak kompatibel. Mungkin file versi lama.")
②

Parsing Header & Ekstraksi Field

Semua field diparsing berdasarkan offset yang sudah diketahui dari format header v3.

offset          = 1
iv              = ciphertext[offset : offset+IV_SIZE];   offset += IV_SIZE
salt            = ciphertext[offset : offset+SALT_SIZE]; offset += SALT_SIZE
ciphertext_body = ciphertext[offset : -MAC_SIZE]   # data terenkripsi
mac_tag         = ciphertext[-MAC_SIZE:]            # 32 byte terakhir
③

Re-derive Kunci (PBKDF2 Ulang)

Dengan SALT dari header dan password dari user, master_key dan mac_key diturunkan ulang — hasilnya identik dengan saat enkripsi selama password benar.

master_key = hashlib.pbkdf2_hmac(
    'sha256', pw, salt, PBKDF2_ITER, dklen=32
)
mac_key = hashlib.sha256(master_key + b'MAC').digest()
④

⚠️ Verifikasi HMAC — Langkah Paling Kritis

MAC diverifikasi menggunakan hmac.compare_digest() yang berjalan dalam waktu konstan (constant-time comparison), mencegah timing attack. Jika MAC tidak cocok, proses dihentikan total — tidak ada byte ciphertext yang diproses. Ini adalah implementasi pola Encrypt-then-MAC yang benar. 

expected_mac = hmac.new(
    mac_key,
    VERSION + iv + salt + ciphertext_body,
    hashlib.sha256
).digest()

if not hmac.compare_digest(expected_mac, mac_tag):
    # HENTIKAN — jangan sentuh ciphertext
    raise ValueError("Autentikasi gagal: file rusak atau kunci salah.")
Mengapa ini penting? Jika dekripsi dilakukan sebelum verifikasi MAC, penyerang bisa mengirim ciphertext rusak berulang kali dan mengamati pesan error ("padding invalid" vs "kunci salah") untuk membocorkan plaintext sedikit demi sedikit — itulah padding oracle attack.
⑤

Dekripsi Chunk & Hapus Padding

Hanya setelah MAC valid, dekripsi dilakukan per chunk dengan operasi terbalik. Padding PKCS#7 dihapus di akhir.

for ci, start in enumerate(range(0, total, CHUNK_SIZE)):
    chunk      = ciphertext_body[start : start+CHUNK_SIZE]
    n          = len(chunk)
    rks, salts = _precompute(master_key, iv, ci, n)
    out       += _dec_chunk(chunk, rks, salts)
    # _dec_chunk: Swap → SubSalt → InvSBox → XOR (urutan terbalik)

return _remove_padding(bytes(out))
# _remove_padding: data[: -data[-1]]

Background Threading

Proses enkripsi/dekripsi dijalankan di thread terpisah agar UI tidak freeze. Komunikasi ke main thread menggunakan self.after(0, fn) — satu-satunya cara aman memodifikasi widget tkinter dari thread lain.

def _start_job(self, label, fn):

    self._btn_enc.configure(state=tk.DISABLED)
    threading.Thread(target=fn, daemon=True).start()

def cb(done, total):
    pct = int(done / total * 100)
    self.after(0, lambda p=pct: self._update_prog(p, ...))

Error Handling Anti-Oracle

Pesan error dirancang generik untuk mencegah kebocoran informasi. Penyerang tidak dapat membedakan "kunci salah" vs "MAC rusak" vs "padding invalid" dari pesan yang ditampilkan.

except ValueError as ex:
    err = str(ex)
    # Tampilkan pesan HMAC/format apa adanya (sudah aman)
    # Sembunyikan detail error lain (padding, unpack, dll)
    if ("autentikasi" not in err.lower() and
        "kompatibel" not in err.lower() and
        "valid" not in err.lower()):
        err = "Kunci salah atau file tidak dapat didekripsi."
    self.after(0, lambda: self._finish(False, err))

Glosarium Istilah Kriptografi & Pemrograman

Kamus lengkap istilah teknis yang digunakan dalam SimpleCipher v3, disertai penjelasan sederhana dan contoh konkret.

A
AES (Advanced Encryption Standard)

Standar enkripsi simetris yang diadopsi oleh pemerintah AS dan digunakan secara global. Menggunakan arsitektur SPN dengan blok 128-bit dan kunci 128/192/256-bit.

Contoh: File yang dienkripsi dengan AES-256 di VeraCrypt atau BitLocker.
Array

Struktur data yang menyimpan kumpulan nilai dengan tipe yang sama dalam urutan berindeks.

Contoh: data = [72, 101, 108, 108, 111] menyimpan 5 byte dalam satu variabel.
Avalanche Effect (Efek Avalanche)

Sifat cipher yang baik di mana perubahan 1 bit pada plaintext atau kunci menyebabkan perubahan ~50% bit pada ciphertext.

Contoh: Mengubah "Halo" menjadi "halo" (H→h) menghasilkan ciphertext yang sama sekali berbeda.
B
Byte

Unit data digital yang terdiri dari 8 bit, dapat merepresentasikan nilai 0–255.

Contoh: Huruf 'A' = 65 = 01000001 dalam biner = 1 byte.
Bijektif

Fungsi matematika yang bersifat satu-ke-satu dan onto — setiap input punya output unik dan bisa dibalik.

Contoh: S-Box SimpleCipher: jika SBOX[5]=128, maka INV_SBOX[128]=5 (tidak ada duplikasi).
Block Cipher

Algoritma enkripsi yang memproses data dalam blok-blok berukuran tetap (misal 16 byte).

Contoh: SimpleCipher memproses data per chunk 4 MB, tapi operasi internal per blok 16 byte.
Brute-force Attack

Serangan dengan mencoba semua kemungkinan kunci/password sampai menemukan yang benar.

Contoh: Mencoba password "a", "b", "c", … "aa", "ab", … sampai menemukan yang tepat.
C
Ciphertext

Data yang sudah dienkripsi, tampak acak dan tidak bisa dibaca tanpa kunci dekripsi.

Contoh: b'\x3f\x8a\x12\x9c...' adalah ciphertext dari plaintext "Halo".
Chunking

Teknik membagi file besar menjadi potongan-potongan berukuran tetap untuk efisiensi memori.

Contoh: File 10 MB diproses menjadi 3 chunk: 4 MB + 4 MB + 2 MB.
Confusion (Konfusi)

Prinsip kriptografi Shannon yang membuat hubungan antara kunci dan ciphertext menjadi sangat kompleks.

Contoh: S-Box di SimpleCipher mengubah byte secara non-linear sehingga sulit ditebak hubungannya dengan kunci.
Constant-time Comparison

Teknik perbandingan yang memakan waktu sama terlepas dari seberapa cocok datanya, mencegah timing attack.

Contoh: hmac.compare_digest(mac1, mac2) selalu butuh waktu sama, baik cocok maupun tidak.
Callback

Fungsi yang dipanggil kembali setelah suatu operasi selesai, biasanya untuk update UI.

Contoh: progress_cb(done, total) dipanggil setiap chunk selesai untuk update progress bar.
Chosen-ciphertext Attack

Serangan di mana penyerang bisa memilih ciphertext dan melihat hasil dekripsinya untuk membocorkan kunci.

Contoh: Penyerang mengirim file terenkripsi yang dimodifikasi ke server dan mengamati error yang dihasilkan.
CSPRNG

Cryptographically Secure Pseudo-Random Number Generator — generator angka acak yang aman secara kriptografis, tidak bisa diprediksi.

Contoh: os.urandom(16) menghasilkan 16 byte acak untuk IV yang tidak bisa ditebak.
D
Dekripsi

Proses mengubah ciphertext kembali menjadi plaintext menggunakan kunci yang benar.

Contoh: decrypt(b'\x3f\x8a...', 'password') → b'Halo'.
Diffusion (Difusi)

Prinsip kriptografi Shannon yang menyebarkan pengaruh setiap bit plaintext ke banyak bit ciphertext.

Contoh: Operasi Swap Halves di SimpleCipher memindahkan byte dari kiri ke kanan agar pengaruhnya menyebar.
Domain Separation

Teknik menggunakan label berbeda untuk memastikan kunci yang diturunkan tidak saling tumpang-tindih.

Contoh: master_key + b'MAC' memastikan mac_key berbeda dari round_key meski dari sumber yang sama.
Differential Cryptanalysis

Teknik analisis kriptografi yang mempelajari bagaimana perbedaan input mempengaruhi perbedaan output.

Contoh: Menganalisis bagaimana mengubah 1 bit di plaintext mengubah pola bit di ciphertext untuk mencari kelemahan S-Box.
E
Enkripsi

Proses mengubah plaintext menjadi ciphertext menggunakan algoritma dan kunci.

Contoh: encrypt(b'Halo', 'password') → b'\x3f\x8a\x12...'.
Encrypt-then-MAC (EtM)

Pola keamanan di mana data dienkripsi dulu, lalu MAC dihitung atas ciphertext.

Contoh: SimpleCipher v3: enkripsi chunk → gabung → hitung HMAC → simpan (bukan MAC dulu baru enkripsi).
Enkripsi Simetris

Sistem enkripsi yang menggunakan kunci sama untuk enkripsi dan dekripsi.

Contoh: SimpleCipher menggunakan password yang sama untuk mengenkripsi dan mendekripsi file.
F
Fisher-Yates Shuffle

Algoritma pengacakan array yang menghasilkan permutasi acak seragam.

Contoh: _rng.shuffle(SBOX) mengacak nilai 0–255 menjadi urutan acak untuk S-Box.
G
GUI (Graphical User Interface)

Antarmuka pengguna berbasis grafis dengan tombol, jendela, dan elemen visual.

Contoh: Jendela SimpleCipher dengan tombol "Browse", "Enkripsi", dan progress bar.
H
Hash

Fungsi yang mengubah input berukuran berapa pun menjadi output berukuran tetap (sidik jari digital).

Contoh: SHA256("Halo") → 32 byte tetap. Sedikit perubahan input mengubah total output.
HMAC (Hash-based Message Authentication Code)

Kode autentikasi pesan yang menggunakan fungsi hash dan kunci rahasia untuk verifikasi integritas.

Contoh: HMAC-SHA256(mac_key, ciphertext) → 32 byte tag untuk memverifikasi file tidak dimodifikasi.
I
IV (Initialization Vector / Nonce)

Nilai acak yang digunakan sekali per sesi enkripsi untuk memastikan ciphertext unik meski plaintext sama.

Contoh: Enkripsi file "Halo.txt" dua kali menghasilkan ciphertext berbeda karena IV berbeda.
Iterasi

Pengulangan proses tertentu sebanyak jumlah yang ditentukan.

Contoh: PBKDF2 dengan 10.000 iterasi berarti proses hash diulang 10.000 kali untuk memperlambat brute-force.
K
Key Derivation Function (KDF)

Fungsi yang menurunkan kunci kriptografis dari password atau passphrase.

Contoh: PBKDF2 mengubah password "rahasia123" menjadi master_key 32 byte yang aman.
Kriptografi

Ilmu dan seni menjaga kerahasiaan informasi dengan mengubah data menjadi bentuk yang tidak bisa dibaca tanpa kunci.

Contoh: Mengirim pesan rahasia dengan enkripsi sehingga hanya penerima yang punya kunci bisa membaca.
L
Linear Cryptanalysis

Teknik analisis yang mencari hubungan linear antara plaintext, ciphertext, dan kunci.

Contoh: Mencari persamaan seperti P[2] ⊕ C[5] ⊕ K[3] = 0 yang berlaku dengan probabilitas tinggi.
Lookup Table

Tabel yang digunakan untuk mencari nilai pengganti secara cepat.

Contoh: SBOX[128] = 45 artinya byte bernilai 128 diganti dengan 45.
M
MAC (Message Authentication Code)

Kode pendek yang ditambahkan pada pesan untuk memverifikasi keaslian dan integritas.

Contoh: HMAC-SHA256 menghasilkan MAC 32 byte yang ditempel di akhir file terenkripsi.
Modulo

Operasi matematika yang menghasilkan sisa bagi.

Contoh: (200 + 80) % 256 = 24 (280 dibagi 256 sisa 24).
Master Key

Kunci utama yang diturunkan dari password, digunakan untuk menghasilkan kunci-kunci lain.

Contoh: PBKDF2 menghasilkan master_key 32 byte, lalu master_key ini menurunkan round_key dan mac_key.
N
Non-linear

Operasi atau fungsi yang tidak mengikuti pola garis lurus, sulit diprediksi dengan persamaan linear.

Contoh: S-Box di SimpleCipher: input 5 → output 128, input 6 → output 45 (tidak ada pola linear).
NumPy

Library Python untuk komputasi numerik dengan array multidimensi yang cepat.

Contoh: _NP_SBOX[data] melakukan lookup S-Box untuk jutaan byte sekaligus dalam hitungan milidetik.
P
Padding (PKCS#7)

Penambahan byte di akhir data agar panjangnya kelipatan ukuran blok tertentu.

Contoh: Data 13 byte ditambah 3 byte bernilai 0x03 menjadi 16 byte.
PBKDF2 (Password-Based Key Derivation Function 2)

Fungsi standar untuk menurunkan kunci dari password dengan iterasi berulang dan salt.

Contoh: PBKDF2-HMAC-SHA256(password, salt, 10.000 iterasi) → master_key 32 byte.
Plaintext

Data asli yang belum dienkripsi, bisa dibaca langsung.

Contoh: b'Halo Dunia' adalah plaintext sebelum dienkripsi.
Progress Bar

Indikator visual yang menunjukkan persentase penyelesaian suatu proses.

Contoh: Bar biru di SimpleCipher yang bergerak dari 0% ke 100% saat enkripsi berlangsung.
R
Rainbow Table

Tabel prekomputasi yang berisi hash dari jutaan password untuk mempercepat cracking.

Contoh: Tabel yang sudah menghitung hash "password123", "123456", dll sehingga penyerang tinggal lookup.
Round Key (Kunci Ronde)

Kunci turunan yang berbeda untuk setiap ronde enkripsi.

Contoh: SimpleCipher punya 6 round key (RK₀ sampai RK₅), masing-masing sepanjang data.
Ronde (Round)

Satu siklus lengkap operasi transformasi dalam cipher blok.

Contoh: SimpleCipher melakukan 6 ronde, setiap ronde: XOR → S-Box → AddSalt → Swap.
S
S-Box (Substitution Box)

Tabel lookup yang mengganti setiap byte dengan byte lain secara non-linear.

Contoh: SBOX[0]=214 artinya byte bernilai 0 diganti jadi 214.
Salt

Nilai acak yang ditambahkan ke password sebelum proses key derivation untuk mencegah rainbow table.

Contoh: Password "rahasia" + salt "x9k2m" di-hash, salt disimpan di header file.
Self-inverse

Operasi yang jika diterapkan dua kali mengembalikan nilai awal.

Contoh: XOR: (data ⊕ key) ⊕ key = data. Swap: swap dua kali = kembali ke posisi semula.
SHA-256

Fungsi hash kriptografis yang menghasilkan output 32 byte (256 bit).

Contoh: SHA256("Halo") → b'\x3f\x8a\x12...' (32 byte).
SPN (Substitution-Permutation Network)

Arsitektur cipher blok yang menggabungkan lapisan substitusi (S-Box) dan permutasi (swap).

Contoh: SimpleCipher terinspirasi AES, menggunakan 6 ronde SPN: substitusi via S-Box, permutasi via Swap Halves.
T
Threading

Teknik menjalankan beberapa proses secara paralel dalam satu program.

Contoh: SimpleCipher menjalankan enkripsi di background thread agar UI tidak freeze.
Timing Attack

Serangan yang menganalisis waktu eksekusi untuk membocorkan informasi rahasia.

Contoh: Jika perbandingan MAC lebih cepat saat byte pertama cocok, penyerang bisa menebak MAC byte per byte.
Tkinter

Library GUI bawaan Python untuk membuat aplikasi desktop.

Contoh: Jendela SimpleCipher dibuat dengan tk.Tk(), tk.Button(), tk.Entry().
W
Work Factor

Jumlah usaha komputasi yang diperlukan untuk menyerang sistem, biasanya diukur dalam iterasi atau waktu.

Contoh: PBKDF2 dengan 10.000 iterasi berarti work factor 10.000× lebih berat dari hash tunggal.
X
XOR (Exclusive OR)

Operasi bitwise yang menghasilkan 1 jika bit berbeda, 0 jika bit sama.

Contoh: 01001000 ⊕ 10110101 = 11111101 (72 ⊕ 181 = 253).
